--- section: contribute lang: en title: Making use of the Glossary authors: - Sam Smith ---

The Open Data Handbook has a glossary page, for defining uncommon, or specialised terms.

When using such terms you should:

  1. Check if they already exist in the glossary
  2. If they don’t exist, add them
  3. Link to the terms from your text

We’ll look at how to do steps 2 and 3 below.

Adding a term to the Glossary

The glossary is maintained in a similar way to the Handbook menu, so if you’ve read the adding a page section, the process should be quite familiar.

To get started, navigate to the handbook-glossary-en.yml file on Github, which can be found at https://github.com/{{ site.github_username }}/{{ site.github_repo }}/blob/gh-pages/_data/handbook-glossary-en.yml

Here you will see a list of terms, followed by definition, see, link or a combination thereof. Let’s have a quick look at what these keywords mean in this context.

Term

The word or phrase that you are defining. This must always be included.

Definition

The description of the word or phrase, which you wish to be displayed after the term. This should always be included, unless the definition is a duplicate of another in the glossary, in which case use see.

See

Adding a see value will include a link to another term in the glossary. This is useful if your term is defined elsewhere, in an acronym of the term for example. The value you define here must be the same as one of the term values in the document.

Link

The definitions in the glossary are only designed to accommodate a single paragraph of text. If you wish to provide a more detailed explanation, you should link to it elsewhere. A link value will generate a ‘read more’ link, which points to the URL you define here. A link value should therefore be a full URL, i.e http://opendefinition.org/

OK, to start adding terms, first of all switch to the editable view on Github [pencil icon]. Now find where in the list your term should go (it’s alphabetical) and create a new line.

In this example I want to add a definition for the term ‘Open Definition’, so I’m going to slot it in between ‘Open Data’ and ‘Open Government Data’

    - term: "Open Data"
      definition: "Open data are able to be used for any purpose."
      link: “http://opendefinition.org"

    - term: "Open Government Data"

Following the syntax of the document, I simply need to add my entry like this:

    - term: "Open Data"
      definition: "Open data are able to be used for any purpose."
      link: “http://opendefinition.org"
    - term: "Open Definition"
      definition: "The Open Definition sets out principles that define “openness” in relation to data and content."
    - term: "Open Government Data"

At this stage you might be happy and ready to commit your changes. As mentioned above though, there are a couple of other options available here. Perhaps in the above example it might be good to reference ‘Open Data’, as the two are quite related. To do that just a line like so:

    - term: "Open Definition"
      definition: "The Open Definition sets out principles that define “openness” in relation to data and content."
      see: "Open Data"

Better than that though, would probably be to link to the Open Definition itself, we can do that by adding a link value:

    - term: "Open Definition"
      definition: "The Open Definition sets out principles that define “openness” in relation to data and content."
      link: "http://opendefinition.org/od/"

Now the outputted text on the site will read:

Open Definition
The Open Definition sets out principles that define “openness” in relation to data and content. Read more.

Perfect. You are ready to commit and make a pull request (see Adding a Page).

Linking to a Glossary term

To reference a word or phrase in the glossary, you can link directly to that term on the page, you just need a little know-how.

The key piece of information you need to know is the ID of the term you are linking to. This will be the term as it appears in handbook-glossary-en.yml, all in lower case, and any spaces replaced with hyphens (-).

For example, the ID for the term ‘Open Definition’ would be open-definition.

To look at what we do with this ID, let’s go through an example. Here is a little text, written in Markdown:

The full Open Definition gives precise details as to what this means. To summarize the most important:

 * Availability and Access: the data must be available as a whole and at no more than a reasonable reproduction cost, preferably by downloading over the internet. The data must also be available in a convenient and modifiable form.
 * Reuse and Redistribution: the data must be provided under terms that permit reuse and redistribution including the intermixing with other datasets.
 * Universal Participation: everyone must be able to use, reuse and redistribute - there should be no discrimination against fields of endeavour or against persons or groups. For example, ‘non-commercial’ restrictions that would prevent ‘commercial’ use, or restrictions of use for certain purposes (e.g. only in education), are not allowed.

To link ‘Open Definition’ in the first line, to its entry in the glossary, we edit it like so:

The full [Open Definition](/en/glossary/#open-definition) gives precise details as to what this means. To summarize the most important:

As you can see, we have added a standard Markdown link to the Glossary page, with the address appended with a hash (#) then the aforementioned ID (open-definition).
